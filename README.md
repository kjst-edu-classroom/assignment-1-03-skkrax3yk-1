# while ループと標準入力

## 今日の目標

- `while True` と `break` で「終わるまで繰り返す」処理が書ける
- `try-except EOFError` で標準入力の終わりを受け取れる
- 1行ずつのテキストに対して、フィルター・変換・集計の処理が書ける
- ループの中で値を覚えておく **状態変数** が使える

## 標準入力の使い方

このセッションのスクリプトは、すべて**標準入力**から1行ずつデータを受け取ります。
ファイルの内容を流し込むには、`cat ... | ...` というパイプの形を使います。
**「ファイルの中身が左から右へ流れて、右側のスクリプトが受け取る」** イメージです。

```bash
cat samples/sample_text.txt | uv run ex08.py
```

この形は macOS / Linux でも Windows PowerShell でも同じように動きます
（PowerShell の `cat` は `Get-Content` の別名なので使えます）。

`samples/` ディレクトリには各課題で使えるサンプルデータが入っています。

> 補足: macOS / Linux では `uv run ex08.py < samples/sample_text.txt` という
> リダイレクトの書き方もあります（PowerShell だとそのままでは動きません）。
> 教材では `cat ... |` の形に統一します。

## 課題の進め方

### スクリプト課題（ex01.py 〜 ex28.py）

スクリプト課題は「同じコンセプトを段階的に書けるようになる」ように作られています。
各コンセプトは次の段階で展開されます:

1. **穴埋め少**（`____` を 2〜3 個埋める）— その回のコンセプトの導入
2. **穴埋め多**（`____` を 4〜5 個埋める）— 構造の練習
3.（難しいコンセプトは段階を増やす）
4. **スクラッチ**（最初から自分で書く）

| グループ | 段数 | ファイル | テーマ |
|---------|-----|---------|--------|
| 1. while + break | 3 | ex01〜03 | `while True` / `break` / 終了文字 |
| 2. try-except EOFError | 4 | ex04〜07 | 標準入力の終わりを受け取る（最重要パターン） |
| 3. フィルター strip | 3 | ex08〜10 | 空行・空白だけの行を削除 |
| 4. フィルター in / not in | 3 | ex11〜13 | 含む語で抽出・除外（`grep` 風） |
| 5. 状態を覚える | 5 | ex14〜18 | 行番号 → 連続空行をまとめる（最難所） |
| 6. 累積（合計）+ replace | 4 | ex19〜22 | 累積合計、カンマ削除、金額合計 |
| 7. 累積の応用（平均） | 3 | ex23〜25 | 件数・平均・0除算ガード |
| 8. split + リスト[i] | 3 | ex26〜28 | 行を空白で分ける、アクセスログ抽出 |

各 ex??.py の冒頭に解説と実行例があります。
**段階的に難しくなる**ので、ex01 から順に進めてください。

実行方法（穴埋めを埋めたあと）:

```bash
uv run ex01.py
# 入力待ちになるので、何か入力 → Enter → q を入力で終了

cat samples/sample_text.txt | uv run ex04.py
# ファイルの中身を標準入力として流し込む
```
